# Python 正则 `re` 包核心知识

## 1. `re` 是什么

`re` 是 Python 标准库里的正则表达式模块，用来做以下几类事情：

- 判断字符串是否满足某种格式
- 从字符串里提取目标内容
- 批量查找多个匹配项
- 对字符串做替换、切分、清洗

最常见场景：

- 提取菜品 ID、订单号、手机号、邮箱
- 校验输入格式
- 从一大段文本里抓指定字段
- 批量替换无效字符

## 2. 最常用的几个函数

### 2.1 `re.match()`

从字符串开头开始匹配。

```python
import re

result = re.match(r"\d+", "123abc")
```

特点：

- 只检查开头
- 开头不匹配就返回 `None`

### 2.2 `re.search()`

扫描整个字符串，找到第一个匹配。

```python
result = re.search(r"\d+", "abc123xyz")
```

特点：

- 最常用
- 适合“从一段文本中提取一个值”

### 2.3 `re.findall()`

找出所有匹配项，直接返回列表。

```python
result = re.findall(r"\d+", "a1b22c333")
# ['1', '22', '333']
```

特点：

- 适合批量提取
- 如果正则里有分组，返回结果会受分组影响

### 2.4 `re.finditer()`

找出所有匹配项，返回迭代器，每一项是 `Match` 对象。

```python
for m in re.finditer(r"\d+", "a1b22c333"):
    print(m.group(), m.start(), m.end())
```

特点：

- 比 `findall()` 更灵活
- 需要位置信息时优先用它

### 2.5 `re.fullmatch()`

要求整个字符串完全匹配。

```python
re.fullmatch(r"\d{11}", "13812345678")
```

特点：

- 适合格式校验
- 比 `^...$` 更直观

### 2.6 `re.sub()`

替换匹配内容。

```python
text = re.sub(r"\s+", " ", "a   b    c")
# "a b c"
```

### 2.7 `re.split()`

按正则切分字符串。

```python
parts = re.split(r"[|,;]", "a|b,c;d")
# ['a', 'b', 'c', 'd']
```

## 3. 推荐先掌握的匹配规则

### 3.1 普通字符

普通字符默认按字面值匹配：

```python
re.search(r"abc", "123abc456")
```

### 3.2 元字符

这些字符有特殊含义：

```text
. ^ $ * + ? { } [ ] \ | ( )
```

如果想匹配它们本身，要转义：

```python
re.search(r"\.", "3.14")
re.search(r"\|", "a|b")
```

### 3.3 常用字符类

- `\d`：数字，等价于 `[0-9]`
- `\D`：非数字
- `\w`：字母、数字、下划线
- `\W`：非 `\w`
- `\s`：空白字符
- `\S`：非空白字符

示例：

```python
re.findall(r"\d+", "订单12共3件")
# ['12', '3']
```

### 3.4 自定义字符集合

```python
[abc]      # a、b、c 中任意一个
[a-z]      # 小写字母
[A-Z]      # 大写字母
[0-9]      # 数字
[^0-9]     # 非数字
```

示例：

```python
re.findall(r"[A-Z]+", "abcIDXYZ")
# ['IDXYZ']
```

## 4. 数量词

- `*`：前一个规则重复 0 次或多次
- `+`：前一个规则重复 1 次或多次
- `?`：前一个规则重复 0 次或 1 次
- `{n}`：恰好 n 次
- `{m,n}`：m 到 n 次

示例：

```python
re.findall(r"\d{2,4}", "1 12 123 12345")
# ['12', '123', '1234']
```

## 5. 贪婪匹配与非贪婪匹配

默认是贪婪匹配，会尽可能多吃字符。

```python
re.search(r"<.*>", "<a>test</a>").group()
# '<a>test</a>'
```

在数量词后面加 `?` 可以改成非贪婪：

```python
re.search(r"<.*?>", "<a>test</a>").group()
# '<a>'
```

常见非贪婪写法：

- `*?`
- `+?`
- `??`
- `{m,n}?`

## 6. 边界匹配

- `^`：字符串开头
- `$`：字符串结尾
- `\b`：单词边界
- `\B`：非单词边界

示例：

```python
re.fullmatch(r"\d{11}", "13812345678")
re.search(r"\bcat\b", "a cat is here")
```

## 7. 分组是核心

### 7.1 捕获分组

用 `()` 表示分组，并且会保存匹配结果。

```python
m = re.search(r"菜品ID:(\d+)\|", text)
print(m.group(1))
```

说明：

- `group(0)`：整个匹配结果
- `group(1)`：第 1 个分组
- `group(2)`：第 2 个分组

### 7.2 多分组

```python
m = re.search(r"(\w+)@(\w+)\.(\w+)", "test@example.com")
```

可以取：

- `m.group(1)` -> 用户名
- `m.group(2)` -> 域名
- `m.group(3)` -> 后缀

### 7.3 命名分组

推荐在结构复杂时使用。

```python
m = re.search(r"菜品ID:(?P<id>\d+)\|菜品名称:(?P<name>[^|]+)", text)
print(m.group("id"))
print(m.group("name"))
```

优点：

- 可读性更高
- 分组多的时候不容易搞错编号

### 7.4 非捕获分组

如果只想分组，不想保存结果：

```python
(?:abc)
```

示例：

```python
re.findall(r"(?:abc)+", "abcabcxyz")
```

## 8. 常见断言

断言用于“判断位置”，自己不消费字符。

### 8.1 正向前查找

```python
exp = r"\d+(?=\|)"
```

意思：右边必须是 `|`，但 `|` 不包含在结果里。

### 8.2 正向后查找

```python
exp = r"(?<=菜品ID:)\d+"
```

意思：左边必须是 `菜品ID:`，但结果只取数字。

### 8.3 组合使用

```python
re.search(r"(?<=菜品ID:)\d+(?=\|)", text).group()
```

这类写法很适合“只要字段值，不要前后标签”。

## 9. `Match` 对象常用方法

正则匹配成功后通常得到一个 `Match` 对象。

常用方法：

- `group()`：整个匹配内容
- `group(n)`：第 n 个分组
- `groups()`：所有分组组成的元组
- `groupdict()`：命名分组组成的字典
- `start()`：匹配起始位置
- `end()`：匹配结束位置
- `span()`：返回 `(start, end)`

示例：

```python
m = re.search(r"菜品ID:(\d+)", text)
print(m.group(0))
print(m.group(1))
print(m.span())
```

## 10. `findall()` 和分组的关系

这是一个非常容易踩坑的点。

如果正则没有分组：

```python
re.findall(r"\d+", "a1b22")
# ['1', '22']
```

如果有一个分组：

```python
re.findall(r"ID:(\d+)", "ID:1 ID:22")
# ['1', '22']
```

如果有多个分组：

```python
re.findall(r"(\w+):(\d+)", "a:1 b:22")
# [('a', '1'), ('b', '22')]
```

结论：

- `findall()` 返回什么，取决于分组结构
- 结构复杂时，`finditer()` 往往更稳

## 11. `compile()` 什么时候用

当同一个正则会反复使用时，建议先编译。

```python
pattern = re.compile(r"菜品ID:(\d+)\|")
m = pattern.search(text)
```

适合：

- 循环中重复匹配
- 多处复用同一个正则
- 想把规则和调用逻辑分开

## 12. 常用标志位

- `re.I` / `re.IGNORECASE`：忽略大小写
- `re.M` / `re.MULTILINE`：多行模式，`^` 和 `$` 对每一行生效
- `re.S` / `re.DOTALL`：让 `.` 可以匹配换行
- `re.X` / `re.VERBOSE`：允许写带注释的可读性正则

示例：

```python
re.search(r"abc", "ABC", re.I)
```

## 13. 原始字符串一定要养成习惯

正则里经常有反斜杠，推荐始终写成原始字符串：

```python
r"\d+\.\d+"
```

而不是：

```python
"\\d+\\.\\d+"
```

原因：

- 更清晰
- 少一层转义
- 减少出错

## 14. 替换时的常见写法

### 14.1 直接替换

```python
re.sub(r"\d+", "NUM", "a1b22")
# 'aNUMbNUM'
```

### 14.2 用分组重组字符串

```python
re.sub(r"(\d{4})-(\d{2})-(\d{2})", r"\3/\2/\1", "2026-04-14")
# '14/04/2026'
```

### 14.3 用函数替换

```python
def double_num(m):
    return str(int(m.group()) * 2)

re.sub(r"\d+", double_num, "a1b22")
# 'a2b44'
```

## 15. 你的这个场景怎么提取菜品 ID

原始字符串：

```text
菜品ID:14|菜品名称:香菇青菜|价格:¥13.00|...
```

推荐写法 1：

```python
re.search(r"菜品ID:(\d+)\|", text).group(1)
```

推荐写法 2：

```python
re.search(r"(?<=菜品ID:)\d+(?=\|)", text).group()
```

两者区别：

- 写法 1 更常见，兼容性好，易读
- 写法 2 结果更干净，但断言对初学者不如分组直观

## 16. 常见坑

### 16.1 直接对 `None` 调 `group()`

```python
m = re.search(r"\d+", "abc")
print(m.group())  # 会报错
```

正确做法：

```python
m = re.search(r"\d+", "abc")
if m:
    print(m.group())
```

### 16.2 忘了转义特殊字符

比如想匹配 `.`、`|`、`(`、`)`，都要注意转义。

### 16.3 `.*` 用得太宽

```python
r"ID:(.*)\|"
```

这种写法容易吃掉过多内容。  
字段型文本更建议用：

```python
r"ID:([^|]+)\|"
```

### 16.4 把 `match()` 当成“全局搜索”

`match()` 只看开头。  
大多数“提取字段”场景应该先考虑 `search()`。

### 16.5 `findall()` 因分组导致结果结构变化

这一点非常常见，尤其在后续遍历处理时容易出错。

## 17. 处理结构化文本的实用原则

面对这种带分隔符的文本：

```text
菜品ID:14|菜品名称:香菇青菜|价格:¥13.00|...
```

建议优先用下面这种思路：

- 定位字段名前缀
- 明确字段值范围
- 以分隔符作为结束条件

例如：

```python
r"菜品名称:([^|]+)\|"
```

比下面更稳：

```python
r"菜品名称:(.*)\|"
```

原因是 `[^|]+` 明确限制“匹配到下一个 `|` 之前为止”。

## 18. 核心记忆点

- 提取单个值优先用 `re.search()`
- 批量提取优先考虑 `re.findall()` 或 `re.finditer()`
- 校验整个字符串是否匹配优先用 `re.fullmatch()`
- 正则字符串尽量写成原始字符串 `r"..."`
- 结构化文本提取字段时，优先用 `[^分隔符]+`，少用宽泛的 `.*`
- 有多个字段时优先用命名分组
- 处理匹配结果前先判断是否为 `None`
- 同一个正则反复使用时用 `re.compile()`
